# ARCHITECTURE.md

The architecture is **non-negotiable**. Changes require an ADR and a ticket. Do not edit this file mid-implementation.

---

## The three layers and their responsibilities

```
┌─────────────────────────────────────────────────────────────┐
│ ui/                                                         │
│   Streamlit pages. Render only. Calls services.             │
│   No business logic. No I/O outside services.               │
├─────────────────────────────────────────────────────────────┤
│ services/                                                   │
│   Orchestration. Combines domain operations + port calls.   │
│   Knows about ports (interfaces), not adapters.             │
├─────────────────────────────────────────────────────────────┤
│ domain/                                                     │
│   Pure Python. Models, business rules, FIFO, tax, thesis.   │
│   ZERO I/O imports. No requests, no file I/O, no streamlit. │
└─────────────────────────────────────────────────────────────┘

         ports/  ← Protocols (interfaces). Domain depends on these.
         adapters/  ← Concrete implementations. Swap freely.
```

---

## The Layer 1 / Layer 2 / Layer 3 boundary

We have three logical concerns that must not bleed into each other:

| Layer | What it owns | Failure mode |
|---|---|---|
| **L1: Book of record** | Lots, transactions, FIFO matching, cost basis | Must work offline. Source of truth. |
| **L2: Live valuation** | Current prices, live FX, unrealized gains | Network-dependent. Cache aggressively. Fail gracefully. |
| **L3: Decision support** | Thesis state, decision gates, tax simulator, behavioural ledger | Layered on top of L1+L2. |

**Rule:** L1 must never import from L2 or L3. If yfinance is down, the book still loads. If thesis taxonomy changes, lots don't move.

---

## The dependency rule (clean architecture)

Dependencies point **inward**:
- `ui/` may import from `services/`
- `services/` may import from `domain/` and `ports/`
- `adapters/` may import from `ports/` and `domain/`
- `domain/` may import from **nothing else in the app**

Enforced by `import-linter` (run in CI).

---

## Hard rules

### Domain layer
- No `requests`, `httpx`, `urllib`, `aiohttp`
- No file I/O (`open`, `Path.read_*`)
- No `streamlit`. `pandas` is allowed but no `st.cache_data`.
- No `datetime.now()` — pass `as_of` explicitly for testability
- All models: Pydantic v2 with `model_config = ConfigDict(frozen=True)` where possible
- All money: `Decimal`, never `float`. Use `Money` value object with currency.

### Services layer
- One service per use case (`compute_positions`, `record_buy`, `simulate_sell`, etc.)
- Services are functions, not classes, unless state is genuinely needed
- All services accept ports as parameters (dependency injection — no globals)

### Adapters layer
- Each adapter implements exactly one port
- Adapters are the only place where `requests`/`yfinance`/`json.dump` may appear
- Each adapter has its own `tests/integration/test_<adapter>.py`

### UI layer
- One file per Streamlit page in `ui/pages/`
- Pages call services, never adapters or domain directly
- All formatting (€, %, dates) goes through `ui/format.py`
- No `if`-branching on business logic in UI files — that belongs in services

---

## The four architectural invariants (the "must always be true" list)

1. **FIFO is replayed from edit-date forward.** Editing a 2023 buy lot triggers recompute of every sell from 2023 onward. The recompute is deterministic and idempotent. Audit trail = git history of `portfolio.json`.

2. **Cost basis EUR is frozen at transaction-date ECB FX rate.** Never re-translated. Stored on the lot.

3. **Live valuation EUR uses current FX.** Computed on-the-fly, never persisted on the lot.

4. **Sparerpauschbetrag is a stateful annual budget.** Tracked per calendar year. Sells in 2026 only consume the 2026 allowance. Loss pot (Verlustverrechnungstopf) carries forward.
